## Overview

This repository contains comprehensive Game Design Documents (GDDs) for two eco-friendly African-themed mobile games:

1. **Eco-Warriors** - A fast-paced water conservation game
2. **Jumanji: African Adventure** - An adventure board game through African landscapes

## Project Goals

- To promote environmental awareness through play
- To celebrate African culture and nature
- To show how games can educate and inspire change

## Why African Eco-Friendly Games?

African-themed games provide a unique opportunity to:
- Celebrate rich African cultural heritage
- Educate players about African ecosystems and wildlife
- Promote environmental conservation awareness
- Create engaging, culturally authentic gaming experiences

## Game 1: Eco-Warriors

### Concept
A fun and educational game where players collect clean water while avoiding pollutants.

### Core Gameplay
- Move left/right to catch clean drops
- Avoid trash, oil, and plastic
- Each level gets faster and more challenging
- **Goal:** Keep the community water clean

### African Identity
- African instruments for music and sound
- African patterns, vibrant earthy colors, and traditional designs
- Gourds and baskets for gameplay
- African proverbs and local wisdom

## Game 2: Jumanji: African Adventure

### Concept
An MVP mobile game inspired by the classic Jumanji, redesigned with African culture and adventure.

### Core Gameplay
- Tap to roll dice and move along the jungle path
- Land on random event tiles: mischievous monkeys, hippos, quicksand traps
- Collect power-ups: magic drum, shield, vine swings
- **Goal:** Reach the hidden village to win first

### African Identity
- Board tiles reflect African landscapes: savanna, rainforest, rivers
- Characters in safari-style clothing with cultural patterns
- Dice decorated like African drums

## MVP Principles

Both games follow these MVP principles:
- **Simple core gameplay** - Easy to learn, hard to master
- **Minimal assets required** - Focus on essential features
- **Playable and testable** - Functional from day one
- **Expandable later** - Built with future enhancements in mind

---

## Technology Stack

Both games are built using:
- **Platform:** HTML5/JavaScript (cross-platform mobile)
- **Rendering:** HTML5 Canvas API for 2D graphics
- **Audio:** Web Audio API for sound effects and music
- **Storage:** LocalStorage for persistence
- **Build Tools:** Webpack, Babel

## Getting Started

Each game has three key documents:

1. **Requirements Document** - User stories and acceptance criteria following EARS patterns
2. **Design Document (GDD)** - Complete technical architecture, components, and systems
3. **Implementation Tasks** - Step-by-step development plan

To implement either game, start with the requirements document, review the design, and follow the implementation tasks in order.
